3. WEATHER DATA SOURCES
============================================================

Use weather data in this order:

1. PAGASA for official advisory context.
2. Open-Meteo for structured numeric JSON data.
3. Store all fetched weather snapshots in weather_logs.
4. Frontend displays saved weather logs from Laravel, not direct API calls.

PAGASA official site:
https://pagasa.dost.gov.ph/

PAGASA products and services:
https://pagasa.dost.gov.ph/products-and-services

PAGASA tropical cyclone bulletins:
https://pagasa.dost.gov.ph/tropical-cyclone/severe-weather-bulletin

Open-Meteo API:
https://open-meteo.com/
https://open-meteo.com/en/docs

Open-Meteo sample idea:
Use latitude/longitude of the barangay.
Fetch:
- temperature
- precipitation/rainfall
- wind speed
- wind direction
- weather code

Backend approach:
- WeatherLogController
- WeatherService or simple private method in controller for version 1
- Laravel Http::get()
- Save response in weather_logs

Do not:
- scrape unstable pages unless there is no alternative
- expose API keys in React
- let the frontend decide trusted weather source logic

9. SIMPLIFIED CODE RULES TO PUT IN EVERY AI PROMPT
============================================================

Paste this into every AI coding prompt:

"Write beginner-friendly code. Do not over-engineer. Do not use advanced architecture unless necessary. Use small React components, simple Laravel controllers, Eloquent queries, Form Requests for validation, and API Resources for JSON. Render repeated UI from arrays using .map(). Do not copy-paste repeated cards or rows. Keep files short and readable. Use pagination and summary endpoints for large data. Match the prototype UI/UX, but do not copy the prototype as one huge file."
